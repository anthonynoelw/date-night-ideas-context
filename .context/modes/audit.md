# Mode: Audit (Quality & Security)

## Objective
Review existing code for vulnerabilities, performance bottlenecks, and violations of project standards.

## Core Instructions
1. **Compliance Check:** Ensure all changes adhere to the GDPR and privacy-first principles mentioned in the `project-overview.md`.
2. **Guideline Adherence:** Verify that the code strictly follows the `guidelines/` for the respective language or framework.
3. **Security First:** Look for common vulnerabilities (OWASP top 10), insecure data storage, and missing rate limiting.
4. **Efficiency:** Identify redundant logic, N+1 problems, or expensive UI rebuilds.

## Instructions
- Provide actionable feedback for every issue found.
- Do not accept "quick fixes" that violate the architectural integrity.
